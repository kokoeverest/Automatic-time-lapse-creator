* Testing rules:
    - Always follow the # Arrange, Act & Assert pattern.
    - Prefer with patch syntax over patch decorator when patching functions.

* Type checking:
    - When making changes always check the .pyi files to see if the changes are compatible with the type hints.

* Documentation:
    - If needed make the necessary updates to the readme.md files so the information is accurate.

* Docstrings:
    - When making changes always update the functions and methods docstrings.
